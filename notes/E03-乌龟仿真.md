## 这节解决什么问题

ROS 中节点之间如何互相发现并传递数据？turtlesim 是最小的可运行 ROS 系统，用它理解"多个独立程序如何通过 topic 异步通信"。

## 核心命令

### 启动 turtlesim 系统

```bash
# 终端 1：启动 ROS Master（整个 ROS 网络的中央注册表）
roscore

# 终端 2：启动乌龟模拟器
rosrun turtlesim turtlesim_node

# 终端 3：启动键盘遥控节点
rosrun turtlesim turtle_teleop_key
```

> **关键认知**：`roscore` 必须先启动。所有 ROS 节点启动时都会向 Master 注册自己，Master 负责撮合 publisher 和 subscriber。没有 Master，节点之间无法发现对方。

### 命令行探测 topic

```bash
# 列出当前所有 topic
rostopic list

# 查看某个 topic 上的消息类型
rostopic type /turtle1/cmd_vel

# 查看消息类型的字段定义
rosmsg show geometry_msgs/Twist

# 实时监听 topic 上的数据
rostopic echo /turtle1/pose

# 查看 topic 发布频率
rostopic hz /turtle1/pose

# 向 topic 发布一条消息（让乌龟动起来）
rostopic pub -1 /turtle1/cmd_vel geometry_msgs/Twist "
linear:
  x: 2.0
  y: 0.0
  z: 0.0
angular:
  x: 0.0
  y: 0.0
  z: 1.8"
```

> `-1` 表示只发一次，去掉 `-1` 加 `-r 10` 可以 10Hz 持续发布。

### 查看系统拓扑

```bash
# 列出所有节点
rosnode list

# 查看某个节点的详细信息（发布的 topic、订阅的 topic、提供的 service）
rosnode info /turtlesim
```

## 第一个发布者节点（C++）

O'Kane 书中 Chapter 3 的标准写法：

### 1. 创建功能包

```bash
cd ~/catkin_ws/src
catkin_create_pkg my_pub_sub roscpp geometry_msgs
```

### 2. 写发布者 `pubvel.cpp`

```cpp
#include <ros/ros.h>
#include <geometry_msgs/Twist.h>

int main(int argc, char **argv) {
    ros::init(argc, argv, "pubvel");
    ros::NodeHandle nh;

    ros::Publisher pub = nh.advertise<geometry_msgs::Twist>(
        "/turtle1/cmd_vel", 1000);

    ros::Rate rate(2);

    while (ros::ok()) {
        geometry_msgs::Twist msg;
        msg.linear.x = 2.0;
        msg.angular.z = 1.8;

        pub.publish(msg);
        rate.sleep();
    }
}
```

**逐行解读**：

| 行 | 含义 |
|---|---|
| `ros::init(...)` | 初始化 ROS 节点，节点名 `pubvel` |
| `ros::NodeHandle nh` | 与 ROS 系统通信的句柄，所有操作都通过它 |
| `nh.advertise<MsgType>(topic, queue_size)` | 声明一个 publisher，`queue_size=1000` 是消息缓冲队列长度 |
| `ros::Rate rate(2)` | 设定循环频率 2Hz |
| `ros::ok()` | 检查节点是否被 SIGINT/Ctrl-C 终止 |
| `pub.publish(msg)` | 发布消息到 topic |
| `rate.sleep()` | 休眠以维持设定频率 |

### 3. 写订阅者 `subpose.cpp`

```cpp
#include <ros/ros.h>
#include <turtlesim/Pose.h>

void poseCallback(const turtlesim::Pose::ConstPtr &msg) {
    ROS_INFO_STREAM("x=" << msg->x << " y=" << msg->y
        << " theta=" << msg->theta);
}

int main(int argc, char **argv) {
    ros::init(argc, argv, "subpose");
    ros::NodeHandle nh;

    ros::Subscriber sub = nh.subscribe(
        "/turtle1/pose", 1000, &poseCallback);

    ros::spin();
}
```

| 行 | 含义 |
|---|---|
| `poseCallback` | 回调函数，每收到一条消息调用一次 |
| `ConstPtr` | 共享指针常量引用，避免拷贝 |
| `nh.subscribe(topic, queue_size, callback)` | 注册订阅者 |
| `ros::spin()` | 进入事件循环，不断检查回调队列 |

> **关键区别**：`ros::spin()` vs `ros::spinOnce()` — `spin()` 会阻塞直到 `ros::ok()=false`；`spinOnce()` 只处理当前队列中积累的回调然后返回，适合你自己有 while 循环的场景。O'Kane 在第 3 章的 spinner 例子中使用了 `ros::spinOnce()` 配合自定义循环。

### 4. CMakeLists.txt

```cmake
cmake_minimum_required(VERSION 2.8.3)
project(my_pub_sub)

find_package(catkin REQUIRED COMPONENTS roscpp geometry_msgs)

include_directories(${catkin_INCLUDE_DIRS})

add_executable(pubvel pubvel.cpp)
target_link_libraries(pubvel ${catkin_LIBRARIES})

add_executable(subpose subpose.cpp)
target_link_libraries(subpose ${catkin_LIBRARIES})
```

### 5. 编译与运行

```bash
cd ~/catkin_ws
catkin_make
source devel/setup.bash

# 先启动 turtlesim，再运行自己的节点
rosrun my_pub_sub pubvel
```

## 关键知识点

### 1. ROS 计算图模型

```
[pubvel] --/turtle1/cmd_vel--> [turtlesim_node]
                                    |
[turtle_teleop_key] --/turtle1/cmd_vel/

[subpose] <--/turtle1/pose-- [turtlesim_node]
```

- **每个长方形是一个 Node（进程）**
- **每条箭头是一个 Topic（命名的数据总线）**
- **Node 之间不直接调用函数，通过 topic 解耦**

### 2. Publisher / Subscriber 是异步的

- Publisher 只管发，不知道有没有人在听
- Subscriber 只管收，不知道消息是谁发的
- 这就是 **publish-subscribe 模式**，与"函数调用"完全不同

### 3. ros::spin() 到底做了什么

```
while (ros::ok()) {
    // 取出回调队列中的所有消息
    // 依次调用对应的 callback 函数
    // 如果队列空，阻塞等待
}
```

## 常见错误

| 错误 | 原因 |
|---|---|
| `[ERROR] [...] Cannot connect to master` | 忘记先启动 `roscore` |
| 编译报 `undefined reference` | CMakeLists.txt 里没加 `target_link_libraries` |
| 运行节点没效果 | 忘记 `source devel/setup.bash` |
| `rostopic pub` 用了 Tab 补全但消息格式不对 | 用 `rostopic pub -1 /topic type` 然后 Tab 自动展开模板数据 |

---

## 一句话总结

**ROS 节点之间通过 topic 进行异步的 publish-subscribe 通信；`roscore` 是注册中心，`ros::spin()` 驱动回调循环；turtlesim 是理解这一切的最小实验平台。**
