## 关键问题及答案

### 1. 为什么要先启动 `roscore`？如果三个终端各跑一个 `roscore`，会发生什么？

**答**：

`roscore` 启动了 **ROS Master**，它是整个 ROS 网络的**中央注册表**。

**作用**：

- 就像手机通讯录 —— 所有节点启动时都要向 Master 注册
- Master 负责**撮合** publisher 和 subscriber，让它们建立直接连接

**如果三个终端各跑一个** **`roscore`**：

- 会产生 **3 个独立的 ROS 网络**
- 终端 A 的节点只能和终端 A 的节点通信
- **它们之间完全隔离**，无法互相发现

### 2. Publisher 发布消息时，知道谁在订阅吗？如果没有任何 subscriber，消息会怎样？

**答**：

**Publisher 不知道谁在订阅**。

ROS 的发布/订阅模型是**松耦合的**：

- Publisher 只管往 topic 发消息
- Subscriber 只管从 topic 收消息
- 它们**不需要知道对方的存在**

**如果没有任何 subscriber**：

- 消息会被**丢弃**（白发了）
- Publisher 不会报错，继续发

### 3. `queue_size=1000` 中的 1000 是什么含义？改成 1 和改成 0 有什么区别？

**答**：

`queue_size` 是**消息缓冲队列的长度**。

| 值      | 含义            | 适用场景                  |
| ------ | ------------- | --------------------- |
| `1000` | 最多缓存 1000 条消息 | subscriber 处理慢时，消息不会丢 |
| `1`    | 只缓存最新 1 条消息   | 传感器数据（如雷达），只关心最新的     |
| `0`    | **无限缓存**（不推荐） | 内存会爆，程序可能崩溃           |

**举例**：

- Publisher 每秒发 100 条消息
- Subscriber 每秒只能处理 10 条
- `queue_size=1000` → 缓存 1000 条后开始丢最老的
- `queue_size=1` → 永远只保留最新的 1 条，中间的都丢

### 4. `ros::spin()` 和 `ros::spinOnce()` 的本质区别是什么？

**答**：

| 函数                | 行为                       | 适用场景                        |
| ----------------- | ------------------------ | --------------------------- |
| `ros::spin()`     | **阻塞**，进入事件循环，一直运行直到节点关闭 | 节点只负责接收消息，没有其他循环逻辑          |
| `ros::spinOnce()` | **非阻塞**，处理一次回调队列后返回      | 节点有自己的 while 循环，需要在循环中间处理消息 |

**如果在一个需要自己控制循环频率的节点里用** **`spin()`**：

- ❌ 程序会**卡在** **`spin()`**，后面的代码永远不会执行
- 你需要用 `ros::spinOnce()` 代替

### 5. 两个节点向同一个 topic `/turtle1/cmd_vel` 同时发消息，乌龟听谁的？

**答**：

**乌龟会同时收到两个消息，后到的覆盖先到的**。

ROS 的 topic 机制：

- 多个 publisher 可以往同一个 topic 发消息
- subscriber 会收到**所有 publisher 的消息**
- 乌龟节点收到消息后，**按时间顺序处理**，后到的消息会覆盖之前的指令

**实际效果**：

- 键盘 teleop 和 pubvel 同时发 → 乌龟会"抽搐"
- 哪个消息先到，乌龟就先执行哪个
- 哪个频率高，哪个就占主导

**解决方案**：

- 用一个 **Topic 优先级管理器**（如 `twist_mux`）来仲裁
- 或者让键盘和 pubvel 发布到**不同的 topic**

## 终端操作记录

### 1. 启动 turtlesim 系统

| 终端   | 命令                                   | 说明            |
| ---- | ------------------------------------ | ------------- |
| 终端 1 | `roscore`                            | 启动 ROS Master |
| 终端 2 | `rosrun turtlesim turtlesim_node`    | 启动乌龟仿真器       |
| 终端 3 | `rosrun turtlesim turtle_teleop_key` | 启动键盘遥控        |

### 2. 查看 Topic 列表

```bash
rostopic list
```

**输出**：

```
/rosout
/rosout_agg
/turtle1/cmd_vel
/turtle1/color_sensor
/turtle1/pose
```

**Topic 说明**：

| Topic                   | 说明                 |
| ----------------------- | ------------------ |
| `/turtle1/cmd_vel`      | 控制乌龟速度和方向（输入）      |
| `/turtle1/color_sensor` | 乌龟当前位置的颜色（输出）      |
| `/turtle1/pose`         | 乌龟的位姿信息（x, y, 角度等） |
| `/rosout`               | ROS 日志聚合节点         |

### 3. 查看消息类型

```bash
rostopic type /turtle1/cmd_vel
```

**输出**：

```
geometry_msgs/Twist
```

### 4. 查看消息定义

```bash
rosmsg show geometry_msgs/Twist
```

**输出**：

```
geometry_msgs/Vector3 linear
  float64 x
  float64 y
  float64 z
geometry_msgs/Vector3 angular
  float64 x
  float64 y
  float64 z
```

**消息结构**：

```
geometry_msgs/Twist
├── linear (线速度)
│   ├── x: 前进速度 (m/s)
│   ├── y: 横向速度
│   └── z: 垂直速度
└── angular (角速度)
    ├── x: 横滚角速度
    ├── y: 俯仰角速度
    └── z: 旋转角速度 (rad/s)
```

### 5. 监听乌龟位姿

```bash
rostopic echo /turtle1/pose
```

**输出**：

```

y: 5.544444561
theta: 0.0
linear_velocity: 0.0
angular_velocity: 0.0
```

**Pose 消息说明**：

| 字段                | 值    | 说明                 |
| ----------------- | ---- | ------------------ |
| x                 | 5.54 | 乌龟在画布上的 x 坐标（中心位置） |
| y                 | 5.54 | 乌龟在画布上的 y 坐标（中心位置） |
| theta             | 0.0  | 朝向角度（弧度）           |
| linear\_velocity  | 0.0  | 当前线速度              |
| angular\_velocity | 0.0  | 当前角速度              |

### 6. 查看位姿发布频率


**输出**：

## 核心概念总结

### 1. ROS 计算图模型

```
Publisher ──topic──> Subscriber
```

- Publisher 和 Subscriber 之间通过 **topic** 解耦
- 双方**不需要知道对方的存在**
- Master 负责撮合

### 2. 消息传递机制

- **异步通信**：Publisher 发消息后立即返回，不等 Subscriber 处理
- **队列机制**：消息先存入队列，Subscriber 从队列中取出
- **频率匹配**：如果 Publisher 比 Subscriber 快，队列满后丢消息

### 3. 数据类型

| 消息类型                 | 用途   | 包名             |
| -------------------- | ---- | -------------- |
| geometry\_msgs/Twist | 速度控制 | geometry\_msgs |
| turtlesim/Pose       | 位姿信息 | turtlesim      |
| turtlesim/Color      | 颜色信息 | turtlesim      |
