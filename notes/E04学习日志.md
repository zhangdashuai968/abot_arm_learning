## 关键问题及答案

### 1. 为什么 `.msg` 文件只需要一行 `float64 real`，就能同时在 C++ 和 Python 里用？谁在背后做翻译？

**答**：

ROS 的消息类型是**语言无关**的。你定义一次 `.msg` 文件，ROS 的构建系统会**自动生成 C++ 和 Python 的类代码**。

**谁在做翻译**：`message_generation` 包里的代码生成器。

**翻译过程**：
```
Complex.msg → message_generation → C++ .h + Python 类
```

### 2. CMakeLists.txt 里 `add_dependencies(pubcomplex ${${PROJECT_NAME}_EXPORTED_TARGETS})` 这行去掉会怎样？为什么有时候能编译成功，有时候报"找不到头文件"？

**答**：

**去掉会怎样**：可能导致编译失败，报错 `fatal error: custom_msgs/Complex.h: No such file or directory`。

**为什么有时候能编译成功**：如果之前编译过一次，`Complex.h` 已经存在于 `devel/` 目录，再次编译时文件已存在，不会报错。

**为什么有时候报错**：ROS 会**并行编译**所有文件。如果没有依赖关系，可能先编译 `pubcomplex.cpp`，此时 `Complex.h` 还没生成。这是典型的**竞态条件**。

**正确编译顺序**：
```
1. 生成 Complex.h ← custom_msgs_EXPORTED_TARGETS
         ↓
2. 编译 pubcomplex.cpp ← 用到 Complex.h
```

### 3. `std_msgs/Header` 中的 `frame_id` 字段有什么用？如果我写 `"base_link"` 和写 `"camera_link"`，对下游节点意味着什么区别？

**答**：

`frame_id` 是**坐标系名称**，告诉下游节点："这条消息里的数据，是**相对于哪个坐标系**测量的"。

| frame_id | 含义 | 举例 |
|----------|------|------|
| `base_link` | 数据相对于机器人中心 | "障碍物在我（机器人中心）前方 2 米" |
| `camera_link` | 数据相对于相机 | "物体在相机前方 1 米" |
| `laser` | 数据相对于激光雷达 | "墙在雷达前方 3 米" |

**如果写错 frame_id**：TF 坐标变换会出错，导航、避障、SLAM 都会计算错误。

### 4. 消息里嵌套另一个自定义消息（如 `custom_msgs/Complex`），编译顺序是怎样的？ROS 怎么知道先编译哪个？

**答**：

**编译顺序**：
1. 先编译被嵌套的消息（Complex.msg → Complex.h）
2. 再编译嵌套它的消息（Circle.msg → Circle.h）
3. 最后编译使用这些消息的 C++ 代码

**ROS 怎么知道**：CMakeLists.txt 中的 `generate_messages(DEPENDENCIES std_msgs)` 声明了依赖，`add_dependencies` 强制指定编译顺序，catkin 构建系统会解析依赖关系自动排序。

### 5. 如果我把 `int32[] values` 改成 `int32[10] values`，在 C++ 层面生成的代码有什么不同？

**答**：

| 定义 | C++ 类型 | 特点 |
|------|---------|------|
| `int32[] values` | `std::vector<int32_t>` | 不定长，可动态添加/删除 |
| `int32[10] values` | `boost::array<int32_t, 10>` | 定长，固定 10 个元素 |

不定长数组可用 `push_back()`、`clear()`，定长数组只能修改已有元素。

## 核心知识点总结

### 自定义消息 7 步法

| 步骤 | 做了什么 |
|------|---------|
| 1. 创建包 | `catkin_create_pkg custom_msgs roscpp std_msgs message_generation` |
| 2. 定义 .msg | 写消息结构（如 `float64 real`、`float64 imag`） |
| 3. 配 package.xml | 声明 `message_generation`（编译）和 `message_runtime`（运行） |
| 4. 配 CMakeLists.txt | `find_package` + `add_message_files` + `generate_messages` + `catkin_package` |
| 5. 写节点代码 | 发布/订阅消息（`#include <custom_msgs/Complex.h>`） |
| 6. 配可执行文件 | `add_executable` + `target_link_libraries` + `add_dependencies` |
| 7. 编译验证 | `catkin_make` + `source devel/setup.bash` + `rosmsg show` |

### `std_msgs/Header` 三字段

| 字段 | 通俗理解 | 类比 |
|------|---------|------|
| `seq` | 消息编号（自动递增） | 快递单号 |
| `stamp` | 数据产生时间（不是处理时间） | 照片拍摄时间 |
| `frame_id` | 数据在哪个坐标系下 | "以谁为参照物" |

### 类型嵌套

消息里可以包含其他消息类型（内置的或自定义的），像俄罗斯套娃。

### 两种数组

| 类型 | 语法 | C++ 类型 | 适用场景 |
|------|------|---------|---------|
| 不定长 | `int32[]` | `std::vector` | 传感器数据（点数未知） |
| 定长 | `float64[10]` | `boost::array` | 固定数量的参数（如 6 个关节角） |

### 常见错误

| 错误 | 原因 |
|------|------|
| `Complex.h: No such file` | 没加 `add_dependencies` 或没先 `catkin_make` |
| `rosmsg show` 找不到消息 | 忘记 `source devel/setup.bash` |
| `Unknown CMake command "add_message_files"` | `find_package` 没加 `message_generation` |
