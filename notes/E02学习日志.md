## 关键问题及答案

### 1. 主控是什么？工控机里跑的是 Ubuntu 几 + ROS 哪个发行版？

**答**：
- 主控 = NVIDIA Jetson 开发板（Tegra 芯片，检测到 `/dev/ttyTHS*` 设备）
- Ubuntu = 18.04.6 LTS (Bionic Beaver)
- ROS = melodic（Ubuntu 18.04 对应 ROS1 Melodic）
- ros_comm = 1.14.13
- 用户名 = `abot`，主机名 = `robot`
- ROS Master URI = `http://192.168.43.211:11311/`

### 2. 4 个轮子的电机怎么从 ROS topic 一路到达底层电机驱动？

**答**：走电信号链路

```
ROS topic (/cmd_vel)
    ↓
工控机 (NVIDIA Jetson)
    ↓
电信号 (USB转串口)
    ↓
Zeus S2 底盘控制器 (/dev/ttyACM0, HL-340芯片)
    ↓
4 个轮子电机
```

### 3. 机械臂 ROCR6 的控制走什么接口？厂家 SDK 的 ROS 包叫什么名？

**答**：
- 接口 = USB + Ethernet
- ROS 包 = `abot_xarm`（xArm 机械臂控制）
- 完整机械臂相关包 = `robot_arm_description` / `robot_arm_driver` / `robot_arm_grasp` / `robot_arm_moveit` / `robot_arm_planning`

### 4. 深度相机、雷达、IMU 分别接在哪个端口？冲突可能性？

**答**：

| 设备 | 接口类型 | 设备节点 | USB VID:PID | 芯片/厂商 |
|------|---------|---------|-------------|----------|
| 深度相机 | USB 3.0 | - | `2bc5:0614` + `2bc5:0511` | 奥比中光 Orbbec |
| RPLidar A3M12 | USB 转串口 | `/dev/ttyUSB0` 或 `/dev/ttyUSB1` | `10c4:ea60` | CP210x |
| IMU N100 | USB 转串口 | `/dev/ttyUSB0` 或 `/dev/ttyUSB1` | `0483:5740` | STM32F407 |
| Zeus S2 底盘 | USB 转串口 | `/dev/ttyACM0` | `1a86:7523` | HL-340 |

**冲突风险**：RPLidar 和 IMU 可能共用相同 VID/PID 的 USB 转串口芯片，每次插拔 USB 时 `/dev/ttyUSB0` 和 `/dev/ttyUSB1` 可能互换。需通过 udev 规则绑定固定设备名（详见 E09）。

### 5. 系统冷启动到能跑 demo，要敲哪几条命令？

**答**：
```bash
source ~/.bashrc && cd ~/catkin_ws && catkin_make && source devel/setup.bash && roslaunch abot bringup.launch
```

## 完整启动流程

### 1️⃣ 上电
- 打开主电源开关
- 检查电池电压指示
- 确认急停按钮已释放

### 2️⃣ 开机
- 工控机（NVIDIA Jetson）启动
- 进入 Ubuntu 18.04.6 LTS 系统
- 默认账号：`abot`，主机名：`robot`

### 3️⃣ ROS 起来
```bash
# 步骤 1：加载环境变量
source ~/.bashrc

# 步骤 2：进入工作空间
cd ~/catkin_ws

# 步骤 3：编译
catkin_make

# 步骤 4：设置环境
source devel/setup.bash

# 步骤 5：启动整机
roslaunch abot bringup.launch
```

### 4️⃣ 看到话题
另开一个终端，运行：
```bash
rostopic list        # 查看所有话题
rostopic hz /scan    # 检查雷达数据发布率
rqt_graph            # 查看节点通信图
```
