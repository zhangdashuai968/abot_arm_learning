## 这节解决什么问题

Topic 是异步单向的。"请帮我计算 3+5，然后把结果给我" 这种**一问一答**的场景，Topic 不适合。你需要 **Service**。

---

## Topic vs Service（O'Kane 的基本对比）

| 维度 | Topic | Service |
|---|---|---|
| 通信模式 | 发布-订阅（异步单向） | 请求-响应（同步双向） |
| 类比 | 广播电台 | 函数调用 / HTTP 请求 |
| 谁主动 | Publisher 主动发 | Client 主动请求 |
| 是否有返回值 | 无 | 有（Response） |
| 应用场景 | 传感器数据流、速度指令 | 查询状态、触发动作、计算 |
| 持续连接 | 长期 | 每次请求建立一次连接 |

> O'Kane 的经典描述：**"If you have a node that needs something done and also needs to know when it's finished, a service is probably the right choice."**

## Service 定义文件（.srv）

创建 `srv/AddTwoInts.srv`：

```
int64 a
int64 b
---
int64 sum
```

> `---` 是分隔线：**上面是 Request，下面是 Response**。

## 配置 CMakeLists.txt

```cmake
# 1. find_package 加入 message_generation
find_package(catkin REQUIRED COMPONENTS
    roscpp
    message_generation
)

# 2. 声明 .srv 文件
add_service_files(
    FILES
    AddTwoInts.srv
)

# 3. 生成服务代码
generate_messages(
    DEPENDENCIES
    std_msgs
)
```

> 注意：`.srv` 和 `.msg` 共用 `message_generation` 和 `generate_messages`，不需要额外的包。

## 编写 Service Server（C++）

O'Kane Chapter 5 中的标准 service server 模式：

```cpp
#include <ros/ros.h>
#include <custom_srvs/AddTwoInts.h>

// 回调函数签名：Request &req, Response &res → bool
bool add(custom_srvs::AddTwoInts::Request  &req,
         custom_srvs::AddTwoInts::Response &res) {
    res.sum = req.a + req.b;

    ROS_INFO_STREAM("request: " << req.a
        << " + " << req.b << " = " << res.sum);

    return true;  // 返回 true 表示服务执行成功
}

int main(int argc, char **argv) {
    ros::init(argc, argv, "add_two_ints_server");
    ros::NodeHandle nh;

    // advertiseService(service_name, callback)
    ros::ServiceServer server =
        nh.advertiseService("add_two_ints", &add);

    ROS_INFO("Ready to add two ints.");

    ros::spin();  // 等待 client 请求
}
```

**Server 端的核心要素**：

| 要素 | 说明 |
|---|---|
| 回调签名 | `bool callback(Request &req, Response &res)` |
| 返回值 | `true` = 服务成功执行，`false` = 失败 |
| `advertiseService(name, callback)` | 注册服务，向 Master 登记 |
| `ros::spin()` | 进入事件循环，等待服务请求 |

> O'Kane 特别指出：**当你 `return false` 时，client 端的 `call()` 不会抛出异常，但是 `call()` 的返回值也会是 `false`**，你需要检查它。

## 编写 Service Client（C++）

```cpp
#include <ros/ros.h>
#include <custom_srvs/AddTwoInts.h>

int main(int argc, char **argv) {
    ros::init(argc, argv, "add_two_ints_client");
    ros::NodeHandle nh;

    // serviceClient<ServiceType>(service_name)
    ros::ServiceClient client =
        nh.serviceClient<custom_srvs::AddTwoInts>("add_two_ints");

    custom_srvs::AddTwoInts srv;
    srv.request.a = 3;
    srv.request.b = 5;

    // call() 是阻塞调用，直到 server 返回
    if (client.call(srv)) {
        ROS_INFO_STREAM("Sum: " << srv.response.sum);
    } else {
        ROS_ERROR("Failed to call service add_two_ints");
    }
}
```

| 要素 | 说明 |
|---|---|
| `nh.serviceClient<Type>(name)` | 创建 client 对象 |
| `client.call(srv)` | **阻塞**直到收到响应或超时 |
| `srv.request.xxx` | 设置请求参数 |
| `srv.response.xxx` | 读取响应结果 |

## 等待 Service 就绪

如果 server 还没启动，client 直接 `call()` 会失败。两种处理方式：

```cpp
// 方法 1：等待 server 出现（阻塞直到超时或 server 上线）
ros::service::waitForService("add_two_ints", ros::Duration(5.0));

// 方法 2：检查 service 是否存在
bool exists = ros::service::exists("add_two_ints", false);
```

O'Kane 建议在启动脚本里先把依赖的 service 都等好，避免竞争条件。

## 完整 CMakeLists.txt

```cmake
cmake_minimum_required(VERSION 2.8.3)
project(custom_srvs)

find_package(catkin REQUIRED COMPONENTS roscpp message_generation)

add_service_files(FILES AddTwoInts.srv)
generate_messages(DEPENDENCIES std_msgs)

catkin_package(CATKIN_DEPENDS roscpp message_runtime)

include_directories(${catkin_INCLUDE_DIRS})

add_executable(add_two_ints_server src/add_two_ints_server.cpp)
target_link_libraries(add_two_ints_server ${catkin_LIBRARIES})
add_dependencies(add_two_ints_server ${${PROJECT_NAME}_EXPORTED_TARGETS})

add_executable(add_two_ints_client src/add_two_ints_client.cpp)
target_link_libraries(add_two_ints_client ${catkin_LIBRARIES})
add_dependencies(add_two_ints_client ${${PROJECT_NAME}_EXPORTED_TARGETS})
```

## 为什么不用 Service 传高频数据

O'Kane 的一个核心警告：

- Service 的 `call()` 是**阻塞**的，client 等待响应期间什么都不能做
- 每次 Service 调用都是一次**完整的请求-响应往返**，开销比 topic 大得多
- 如果你需要 30Hz 的传感器数据，用 topic；如果你需要"查一次数据库"，用 service

**决策口诀**：
- 数据流 → **Topic**
- 远程过程调用 → **Service**
- 需要进度反馈的长时间任务 → **Action**（见 E06）

## 常见错误

| 错误 | 原因 |
|---|---|
| `call()` 返回 false | server 还没就绪、service 名拼错、或回调返回了 false |
| `rosservice call` 报错 | 参数格式不对，用 Tab 先展开模板再填 |
| 编译报找不到 Request/Response 类型 | 没加 `add_dependencies` |
| srv 文件修改后不生效 | 需要重新 `catkin_make`，生成代码在 `devel/include` 下 |

---

## 一句话总结

**Service 是 ROS 中的同步 RPC 机制：定义 `.srv` 文件 → Server 提供回调 → Client 阻塞调用 → 获得 Response；与 Topic 的互补关系是 ROS 通信模型的核心思想。**
