# 学习路线图（6 阶段，\~12-15 个月）

## 设计原则

- **被动学习 → 主动复刻 → 独立设计**：先按官方 32 个实验走，再做原创 launch，最后给 SCI 课题做整机方案
- **每阶段必须有"动手产出"**：跑通一个实验不算产出，能讲清原理 + 改一个参数看变化才算
- **调优穿插每个阶段**：每阶段至少做一次"改一个参数看变化"实验
- **3-4h/周**（嵌套在 SCI 论文项目时间盒内）：*不另开预算，作为 SCI 硬件熟悉子任务*

## Phase 1 · 认知与 ROS 基础（4-6 周）

**目标**：摸清整车硬件结构 + 系统拓扑 + ROS1 基础通信，能在 Linux + ROS 环境下自由编写小节点。

| 子任务                  | spec                                  |
| -------------------- | ------------------------------------- |
| 本体组成与认知              | `parallel/E01-本体组成与认知.md`             |
| 系统组成与认知              | `parallel/E02-系统组成与认知.md`             |
| ROS 基础 7 件套（E03-E09） | `parallel/P1-03-ROS基础7件套.md`（合并 spec） |

**产出**：

- `notes/E01学习日志.md` + 一张手绘/draw\.io 整车结构图（机械/电气/传感器）
- `parallel/E03-E09` 系列 7 份详细笔记（基于 O'Kane《A Gentle Introduction to ROS》，含完整 C++ 代码示例）
- `cheatsheets/启动链路.md` 雏形（待 Phase 2 完善）

**验收**：能在白纸上画出整车数据流（传感器 → 主控 → 执行器），并能徒手写一个发布/订阅节点。

**目标**：能用代码精确控制底盘移动到指定坐标 + 让机械臂末端到达指定位姿，对正逆解能"看公式不查书"。

| 子任务            | spec                        |
| -------------- | --------------------------- |
| E10 ROS 控制底盘移动 | `parallel/E10-ROS控制底盘移动.md` |
| E11 运动学正解      | `parallel/E11-运动学正解.md`     |
| E12 运动学逆解      | `parallel/E12-运动学逆解.md`     |
| E13 本体结构建模认知   | `parallel/E13-本体结构建模.md`    |
| E14 避障传感器数据读取  | `parallel/E14-避障传感器.md`     |

**产出**：

- `notes/正逆解推导.md`（DH 参数表 + 自己推一遍 ROCR6 的 4×4 变换）
- 一段视频/录屏：底盘走出"L 形" + 机械臂从 home 到指定 (x,y,z,r,p,y)
- `cheatsheets/底盘控制接口.md` + `cheatsheets/机械臂关节限位.md`

**验收**：被人问"为什么改 DH 表里某行 d 值会导致末端位置偏 X"能不卡壳说出来。

| 子任务                                | spec                               |

| E17-E18 MATLAB 雷达 + 栅格地图（可选）       | `parallel/P3-02-MATLAB激光雷达建图.md`   |
| E21 激光 SLAM 闭环检测                   | `parallel/E21-激光SLAM闭环检测.md`       |
| E22 G2O 图优化                        | `parallel/E22-G2O图优化.md`           |
| E23 Cartographer 多传感器融合            | `parallel/E23-Cartographer融合建图.md` |
| E24 导航与避障                          | `parallel/E24-导航与避障.md`            |

**产出**：

- 一张实验室二维地图（pgm + yaml）

**目标**：从相机标定到训练自定义检测模型，到视觉伺服闭环抓取，整条链路自己跑一遍。

## Phase 5 · SCI 联动 · 茶园采摘扩展（8-12 周）

**目标**：把 Phase 4 的能力换皮成"茶叶嫩芽采摘"，并写若干原创 launch / 节点，作为 SCI 论文实验环节的工程基础。

| 候选方向（必做 + 选做）                                  |
| ---------------------------------------------- |
| **必做**：茶叶嫩芽数据集（自采 / 公开数据集对比）                   |
| **必做**：YOLO + 深度滤波（深度相机 <!-- 2026-05-03 lsusb实测修正 --> 深度图过滤误检）              |
| **必做**：原创 launch：`tea_pluck.launch` 串起检测+伺服+夹取 |
| 选做：与 G3P 二指夹持力度策略（嫩芽不能捏烂）                      |
| 选做：移动 + 采摘联动（车自己开到茶垄前停下）                       |

**产出**：1-2 个原创 launch + 配套节点 + demo 视频 + 一份"采摘成功率初步实验报告"喂给 SCI 论文。

**验收**：能在团队会议 demo "车自己走到桌子前 + 看到目标 + 伸手夹起来"。

***

## Phase 6 · 独立方案设计 · 陆空协同（2-3 月）

**目标**：与无人机 / 机械狗 / SLAM 长期项目联动，给 SCI 课题做"陆空协同采摘"完整方案。

**任务设定**（与 SCI 论文最终章节联动）：

> 茶园场景 → UAV 巡园定位茶垄 + 病害检测 → 通知 AGV 抵达 → ROCR6 + G3P 完成嫩芽采摘 → 收集到框

**产出**：

- `final_proposal.md`：硬件 BOM + 软件架构 + 风险点 + 与现有 abot 整机的差异表
- 与「云深处 Lite2」「万创陆空两栖」的对比表（哪些复用 / 哪些自研 / 为什么）

**验收**：方案能让导师/队长 review 后说"按这个走能落地"。

***

## 风险与节奏

- **风险**：实验场地占用 / SCI 论文冲刺期占满 → 路线图允许暂停，恢复时从最近完成的 spec 续接
- **加速窗口**：寒暑假可压缩为 1.5x
- **失速信号**：连续 3 周未推进任何 `parallel/` spec → 触发 `notes/` 复盘，调整阶段范围或换学习方式
- **冲突管理**：与 SCI 主线时间盒共用，本项目优先级 = SCI 当周阻塞点优先级；不阻塞 SCI 时，按 Phase 顺序推进
