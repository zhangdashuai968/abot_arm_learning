# E02 · 系统组成与认知

**阶段**：Phase 1
**预计耗时**：2-3 小时
**前置**：E01 已完成（看过实物）
**对应官方实验讲义**：`~/桌面/abot开源机器人/v01.01.01.0000/实验2_系统组成与认知实验.md`

## 目标

理解整车的**软件系统拓扑**：主控（工控机）跑什么、各传感器怎么接入、ROS 节点之间怎么通信、电源怎么走。能画出"上电→开机→ROS 起来"的完整启动链路。

## 关键问题

1. 主控是什么？工控机里跑的是 Ubuntu 几 + ROS 哪个发行版（Melodic / Noetic）？
2. 4 个轮子的电机怎么从 ROS topic 一路到达底层电机驱动？(`/cmd_vel` → ? → 底盘控制器 → 电机)
3. 机械臂 ROCR6 的控制走 USB / CAN / 以太网哪种？厂家 SDK 的 ROS 包叫什么名？
4. 深度相机走 USB 3.0；RPLidar A3M12 走串口/USB；IMU N100 走串口——分别接在哪个端口？冲突可能性？
5. 系统冷启动到能跑 demo，要敲哪几条命令？

## 执行步骤

### 1. 登录主控（30 分钟）

- [x] 找到工控机的开机方式 + 默认账号密码（**记下来**，但**不写入此文件、不进 git**）
- [x] `cat /etc/os-release` → Ubuntu 版本
- [x] `echo $ROS_DISTRO` → ROS 版本
- [x] `roscore` 能不能起来
- [x] `ls /dev/tty*` → 看串口设备
- [x] `lsusb` → 看 USB 设备
- [x] `ip addr` → 看雷达/相机的网线 IP（如有）

### 2. 摸 ROS 工作空间（1 小时）

- [x] 找到主工作空间（猜 `~/abot_ws/` 或 `~/catkin_ws/`）
- [x] `ls src/` 看 ROS 包列表
- [x] 重点 grep：`grep -r "package name=" --include=package.xml | head -30`
- [x] 看是否有 launch 文件总入口（如 `abot_bringup`、`complex_robot_bringup`）

### 3. 跑一遍最小启动（如果设备就绪）

```bash

roslaunch <bringup_pkg> <main_launch>.launch
# 另开终端：
rostopic list
rqt_graph
```

把 `rqt_graph` 截图存 `notes/系统拓扑-rqt_graph.png`。

### 4. 电源与紧急停止

- [x] 找到主电源开关位置
- [x] 找到急停按钮
- [x] 确认电池电压指示
