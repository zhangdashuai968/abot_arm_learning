# E24 · 导航与避障（move_base / nav stack）

**阶段**：Phase 3 收官
**预计耗时**：4-5 小时
**前置**：E23 已出一张可用地图（pgm + yaml）
**对应官方实验讲义**：`~/桌面/abot开源机器人/v01.01.01.0000/实验24_导航和避障实验.md`

## 目标

跑通 ROS navigation stack：在 E23 出的地图上 → 启动 AMCL 重定位 → rviz 给目标点 → 车自己开过去 + 中途放障碍物看避障。

## 关键问题

1. move_base 的全局规划器（global planner）和局部规划器（local planner）分工？
2. costmap 的 inflation_radius 单位？太大太小各会怎样？
3. AMCL 蒙特卡洛粒子滤波，粒子数过多过少各影响什么？
4. DWA 与 TEB 两种局部规划器，哪个更适合走窄通道？
5. 导航失败常见原因 Top 5（地图坐标错 / 起点未定位上 / costmap 膨胀过大 / 目标在障碍内 / planner 超时）

## 执行步骤

### 1. 保存地图
```bash
rosrun map_server map_saver -f ~/abot_ws/maps/lab_map
# 出 lab_map.pgm + lab_map.yaml
```

### 2. 启动导航栈
```bash
roslaunch <abot>_navigation navigation.launch map:=$HOME/abot_ws/maps/lab_map.yaml
```

### 3. AMCL 初定位
- rviz 用 "2D Pose Estimate" 给一个粗略起始位姿
- 推车走两步让 AMCL 收敛
- 看粒子云收紧

### 4. 发目标 + 避障测试
- rviz "2D Nav Goal" 设个 5 m 远的点
- 车开过去
- 中途放一个箱子，看是否绕开
- 把目标设到一个被障碍包围的位置，看能否报"无法规划"

### 5. 调 1-2 个参数
- `inflation_radius` 0.3 → 0.6，看走廊通过性
- `max_vel_x` 调慢，看跟踪精度
