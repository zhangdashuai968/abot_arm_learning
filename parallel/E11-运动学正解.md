## 目标

吃透 DH 参数法，能徒手把"6 个关节角"算出"末端 4×4 位姿矩阵"，并用 ROS tf 验证答案。

## 关键问题（执行前先尝试用直觉回答）

1. DH 参数 4 个 (a, α, d, θ) 各是什么意思？为什么不用 6 个独立参数？
2. 标准 DH vs 改进 DH（Modified DH）的差异？ROCR6 用的是哪个？
3. 齐次变换矩阵 4×4 里左上 3×3 是旋转，右上 3×1 是平移，最后一行 0 0 0 1 为什么？
4. 6 个关节角全 0 时，末端在世界系的位置是多少？（home pose）
5. 单位错（角度 vs 弧度）会导致末端位置偏多少？

## 执行步骤

### 1. 抄一份 DH 表
- 从厂家文档 / URDF / 机械臂手册里抄出 ROCR6 的 6 行 DH 参数
- 写入 `notes/正逆解推导.md` 第一节

### 2. 手推一遍正解
- 给一组测试关节角，比如 `[0, 30°, -45°, 0, 60°, 0]`
- 手算 6 个 4×4 矩阵相乘
- 算出末端 (x, y, z, R, P, Y)

### 3. ROS tf 验证
```bash
# 让机械臂走到那组关节角
# 然后查 tf：
rosrun tf tf_echo /base_link /tool0       # 或末端 link 名
```
对比手算结果。

### 4. 写正解函数
用 Python 实现 `forward_kinematics(joint_angles) → 4x4`，与上面手算/tf 三方对比。

## 产出

- `notes/正逆解推导.md` 第一节：
  - DH 表（清晰排版）
  - 手推过程（含 6 个变换矩阵）
  - 与 tf 输出的对比
- 一段 Python 正解函数（占 30 行以内）
