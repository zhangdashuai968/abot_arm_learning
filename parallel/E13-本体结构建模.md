# E13 · 本体结构建模认知（URDF）

**阶段**：Phase 2
**预计耗时**：2-3 小时
**前置**：E11/E12（理解关节与坐标系）
**对应官方实验讲义**：`~/桌面/abot开源机器人/v01.01.01.0000/实验13_本体结构建模认知实验.md`

## 目标

会读 URDF / xacro 文件，能在 rviz 里看到机械臂模型动起来，能改一个 joint limit 看到行为变化。

## 关键问题

1. URDF 与 xacro 的关系？为什么大项目用 xacro？
2. `<link>` 与 `<joint>` 区别？joint 的 type 有几种（revolute / prismatic / fixed / continuous）？
3. `<inertial>` 标签里的惯性张量怎么来的？给错了 Gazebo 仿真会怎样？
4. URDF 里的网格文件（mesh `.stl/.dae`）从哪里来的？能改吗？
5. base_link / base_footprint / world 三个 frame 的层级关系？

## 执行步骤

### 1. 找现有 URDF
```bash
find ~/abot_ws -name "*.urdf*" -o -name "*.xacro" | head -20
roslaunch <description_pkg> display.launch    # 通常厂家给一个 display.launch
```

### 2. rviz 看模型
- joint_state_publisher GUI 拖每个关节的滑块
- 在 tf display 上看每个 link 的坐标系

### 3. 读懂一个关节
- 选 joint 1（基座），看：
  - axis (旋转轴)
  - limit (lower / upper / effort / velocity)
  - origin (相对父 link 的位姿)

### 4. 改一个 limit 看变化
- 把 joint 1 的 upper limit 从 ±360° 改到 ±90°
- 重新 launch + rviz 拖滑块，验证它走不到原来能走的地方
- **改完记得改回**

### 5. URDF 转 SDF（选做）
若有 Gazebo 仿真需求，了解 `gz sdf -p file.urdf` 转换。
