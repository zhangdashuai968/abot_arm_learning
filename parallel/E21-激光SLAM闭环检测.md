# E21 · 激光 SLAM 闭环检测

**阶段**：Phase 3
**预计耗时**：3-4 小时
**前置**：E10（底盘控制）+ 看过雷达 topic
**对应官方实验讲义**：`~/桌面/abot开源机器人/v01.01.01.0000/实验21_激光SLAM闭环检测实验.md`

## 目标

跑一个会做闭环检测的激光 SLAM（如 hector / karto / cartographer），在实验室走一圈回到起点，**看到闭环触发瞬间地图"对齐"的视觉效果**，并能解释闭环检测的判据。

## 关键问题

1. 闭环检测在解决什么问题？没有它会怎样？（地图漂移图示）
2. 闭环检测的"候选筛选"用什么特征？激光的特征比视觉的特征少多少？
3. 假闭环（false positive）的危害比假阴性大还是小？为什么？
4. 闭环触发后，整张图被"拽回去"是怎么实现的？（提示：图优化）
5. 在长走廊 / 大圆环 / 多房间这三种场景，闭环成功率哪个高？

## 执行步骤

### 1. 选一个 SLAM 包
推荐 `karto_slam` 或 `slam_karto`（自带 scan-matching + loop closure）。
```bash
roslaunch slam_karto slam.launch    # 或厂家给的 launch
```

### 2. 故意走一圈
- 用键盘遥控车从 A 出发，走一个大约 5 m × 5 m 的圈，回到 A 附近
- 在 rviz 实时观察地图
- **录 rosbag** 留底：`rosbag record /scan /odom /tf -O loop_test.bag`

### 3. 闭环触发观察
- 在回到起点附近时，看地图有没有"啪"一下对齐
- 如果没有：检查 odom 漂移有多大、SLAM 包的回环参数

### 4. 调一个回环参数
找回环检测阈值（如 `loop_match_minimum_response_coarse`），
- 调宽 → 假闭环增多
- 调严 → 漏闭环
记录前后差异。
