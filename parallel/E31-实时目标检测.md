# E31 · 实时目标检测（部署到 ROS）

**阶段**：Phase 4
**预计耗时**：3-4 小时
**前置**：P4-02 有一个训好的模型
**对应官方实验讲义**：`~/桌面/abot开源机器人/v01.01.01.0000/实验31_实时目标检测实验.md`

## 目标

把 P4-02 训好的 YOLO 模型部署成 ROS 节点，订阅 `/camera/color/image_raw`，实时推理并发布 `BoundingBox` 话题，**目标 ≥ 15 FPS**（实验室工控机 CPU 推理基线）。

## 关键问题

1. CPU 推理 vs GPU 推理 ONNX 的速度差距大概几倍？工控机有没有独显？
2. 置信度阈值 conf=0.25 与 0.5 的取舍？漏检 vs 误检？
3. NMS 是干啥的？iou_threshold 调高调低各影响什么？
4. 图像从 ROS msg → numpy → 模型输入张量的预处理（resize / normalize / BGR→RGB），漏一步会怎样？
5. 推理频率跟不上相机帧率（30 FPS）怎么办？跳帧 / 异步队列 / 降分辨率？

## 执行步骤

### 1. 选推理框架
- 简单：`ultralytics` 自带 `model.predict()` (PyTorch CPU)
- 进阶：ONNX Runtime / OpenCV DNN（更轻）
- 性能：TensorRT（如果有 NVIDIA 卡）

### 2. 写 ROS 节点
```python
# detector_node.py 关键骨架
class Detector:
    def __init__(self):
        self.model = YOLO("best.pt")
        self.bridge = CvBridge()
        rospy.Subscriber("/camera/color/image_raw", Image, self.cb)
        self.pub = rospy.Publisher("/detections", BoundingBoxes, ...)
    def cb(self, msg):
        img = self.bridge.imgmsg_to_cv2(msg, "bgr8")
        results = self.model(img, conf=0.4)
        # 转 BoundingBoxes 消息发布
```

### 3. 可视化
- 在节点里同时发布 `/detections/image` (Image)：原图叠加框
- rviz 或 rqt_image_view 看

### 4. 测帧率
```bash
rostopic hz /detections
```
- 目标 ≥ 15 FPS
- 不达标 → 降分辨率 / 换 yolov8n / 加跳帧

### 5. 写测试 launch
```xml
<launch>
  <include file="$(find realsense2_camera)/launch/rs_camera.launch"/>
  <node name="detector" pkg="abot_vision" type="detector_node.py" output="screen">
    <param name="model_path" value="$(find abot_vision)/models/best.pt"/>
    <param name="conf_threshold" value="0.4"/>
  </node>
</launch>
```
