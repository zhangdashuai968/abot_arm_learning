# P4-02 · 数据集制作 + 模型训练（E29-E30）

**阶段**：Phase 4
**预计耗时**：6-8 小时（合并版，分 2-3 次）
**前置**：E27 已采 50+ 张图
**对应官方实验讲义**：
- 实验 29 数据集制作实验
- 实验 30 模型训练实验

## 目标

走完一遍"采图 → 标注 → 训练 → 评估"完整闭环，产出一个能用的 YOLOv8 模型（先用占位类如"小方块"，**Phase 5 直接换茶叶嫩芽数据集复用此流程**）。

## 关键问题

1. YOLO 数据集的 txt 标注格式：`class_id cx cy w h`，坐标是归一化还是像素？
2. train/val 比例 8:2 vs 7:3 vs 9:1，小样本（<200 张）选哪个？
3. epoch 训多了会过拟合，怎么从 loss 曲线看出来？
4. mAP@0.5 与 mAP@0.5:0.95 区别？业界报哪个？
5. YOLOv8n / s / m / l / x 五档，**部署到 abot 工控机** 选哪个？（看 CPU/GPU 配置）

## 执行步骤

### 1. 标注（E29）
推荐 `labelImg` 或 `labelme`，YOLO 格式输出：
```bash
pip install labelImg
labelImg ~/abot_ws/datasets/raw/
```
- 先标 50 张占位（如红色小方块）
- 输出到 `~/abot_ws/datasets/yolo/{images,labels}/{train,val}/`

### 2. 组织数据集
```yaml
# data.yaml
path: ~/abot_ws/datasets/yolo
train: images/train
val: images/val
names:
  0: red_block
```

### 3. 训练（E30）
```bash
pip install ultralytics
yolo train model=yolov8n.pt data=data.yaml epochs=50 imgsz=640
```

### 4. 看曲线
- `runs/detect/train/results.png`
- 重点：train/val loss 曲线、mAP 曲线
- 如果 val loss 反弹 → 过拟合 → epoch 减半重训

### 5. 导出
```bash
yolo export model=runs/detect/train/weights/best.pt format=onnx
# 后续 E31 用 ONNX Runtime / OpenCV DNN 推理
```
