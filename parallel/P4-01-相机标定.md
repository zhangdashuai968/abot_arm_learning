# P4-01 · 相机标定（内参 + 手眼，E25-E26）

**阶段**：Phase 4
**预计耗时**：3-4 小时（合并版）
**前置**：深度相机 <!-- 2026-05-03 lsusb实测修正 --> 已能 launch + 机械臂能动
**对应官方实验讲义**：
- 实验 25 内参标定实验
- 实验 26 手眼标定实验

## 目标

吃透"内参标定"和"手眼标定"是在标什么变换，能用 ROS 工具自己标一遍并产出 yaml 文件供后续 Phase 4 使用。

## 关键问题

1. 内参矩阵 K 里的 fx, fy, cx, cy 各是什么？为什么 fx ≠ fy（不是正方形像素）？
2. 畸变系数 k1/k2/p1/p2/k3 哪个是径向哪个是切向？
3. 深度相机 <!-- 2026-05-03 lsusb实测修正 --> 出厂自带内参，还需要标吗？什么场景下需要重标？
4. **手眼标定** eye-in-hand 标的是 `T_gripper_camera`（爪子→相机）；eye-to-hand 标的是 `T_base_camera`（基座→相机）。本平台 深度相机 <!-- 2026-05-03 lsusb实测修正 --> 装末端属于哪种？
5. 手眼标定的最少姿态数量？为什么不是 1-2 个就够？

## 执行步骤

### 1. 内参标定（E25）
- 打印一张棋盘格（A4，9×6 内角点，方格 25 mm）
- 跑 `ros-camera_calibration`：
  ```bash
  rosrun camera_calibration cameracalibrator.py --size 9x6 --square 0.025 \
    image:=/camera/color/image_raw camera:=/camera/color
  ```
- 把棋盘格在相机视野里左/右/上/下/远/近/倾斜各动一遍，等 calibrator 进度条满
- COMMIT → 产出 yaml

### 2. 手眼标定（E26）
推荐用 `easy_handeye` 或 `aruco_hand_eye`：
- 把棋盘格 / ArUco 板贴在固定位置
- 让机械臂带相机走 15-20 个不同姿态
- 每个姿态记录 (T_base_gripper, T_camera_target)
- 求解 AX = XB 得 T_gripper_camera

### 3. 验证标定结果
- 在 rviz 同时显示：
  - 机械臂模型（base → gripper 通过 tf）
  - 相机点云（camera_frame → ?）
  - 用标定结果发布 `gripper → camera_frame` 的 static_transform_publisher
- 看点云是否正确出现在视野前方（不是漂浮在天上）
