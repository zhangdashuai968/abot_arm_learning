## ROS 消息系统概览

ROS 的消息类型是语言无关的。你定义一次 `.msg` 文件，ROS 的构建系统会**自动生成 C++ 和 Python 的类代码**。O'Kane 在 Chapter 4 开篇就指出这个核心设计理念。

### 常用内置消息类型

| 类型 | 用途 | 示例字段 |
|---|---|---|
| `std_msgs/String` | 字符串 | `string data` |
| `std_msgs/Int32` | 32 位整数 | `int32 data` |
| `std_msgs/Float64` | 64 位浮点 | `float64 data` |
| `std_msgs/Bool` | 布尔 | `bool data` |
| `std_msgs/Header` | 时间戳 + 坐标系 | `uint32 seq; time stamp; string frame_id` |
| `geometry_msgs/Twist` | 线速度 + 角速度 | `Vector3 linear; Vector3 angular` |
| `geometry_msgs/Pose` | 位置 + 姿态 | `Point position; Quaternion orientation` |
| `sensor_msgs/Image` | 图像 | `Header header; uint32 height; uint32 width; ...` |
| `sensor_msgs/LaserScan` | 激光雷达扫描 | `Header header; float32[] ranges; ...` |

> 用 `rosmsg show 类型名` 可以随时查看完整定义。

## 自定义消息（核心）

### 场景

我们要做一个发布者，发布"复数"（实部 + 虚部），让订阅者接收并打印。

### 1. 创建功能包

```bash
cd ~/catkin_ws/src
catkin_create_pkg custom_msgs roscpp std_msgs message_generation
```

> `message_generation` 是编译自定义消息的必要依赖。

### 2. 定义消息文件

创建 `msg/Complex.msg`：

```
float64 real
float64 imag
```

**命名规范**：消息类型名用 CamelCase（首字母大写），文件名与类型名一致。

### 3. 配置 package.xml

```xml
<build_depend>message_generation</build_depend>
<exec_depend>message_runtime</exec_depend>
```

### 4. 配置 CMakeLists.txt

O'Kane 第 4 章详细解释了以下四个关键修改：

```cmake
find_package(catkin REQUIRED COMPONENTS
    roscpp
    std_msgs
    message_generation       # 1. 添加 message_generation 依赖
)

# 2. 声明要生成代码的 .msg 文件
add_message_files(
    FILES
    Complex.msg
)

# 3. 生成消息代码（列出版本依赖）
generate_messages(
    DEPENDENCIES
    std_msgs
)

# 4. catkin_package 中声明消息运行时依赖
catkin_package(
    CATKIN_DEPENDS roscpp std_msgs message_runtime
)
```

### 5. 发布者节点 `pubcomplex.cpp`

```cpp
#include <ros/ros.h>
#include <custom_msgs/Complex.h>

int main(int argc, char **argv) {
    ros::init(argc, argv, "pubcomplex");
    ros::NodeHandle nh;

    ros::Publisher pub = nh.advertise<custom_msgs::Complex>(
        "complex", 1000);

    ros::Rate rate(1);

    while (ros::ok()) {
        custom_msgs::Complex msg;
        msg.real = 2.5;
        msg.imag = -1.0;

        pub.publish(msg);

        ROS_INFO_STREAM("Sent: " << msg.real
            << " + " << msg.imag << "i");

        rate.sleep();
    }
}
```

> 注意头文件路径是 `<包名/消息类型名.h>`，消息类型域是 `包名::消息类型名`。

### 6. CMakeLists.txt 可执行文件部分

```cmake
add_executable(pubcomplex pubcomplex.cpp)
target_link_libraries(pubcomplex ${catkin_LIBRARIES})

# 关键：确保消息代码先编译
add_dependencies(pubcomplex ${${PROJECT_NAME}_EXPORTED_TARGETS})
```

> O'Kane 特别强调 `add_dependencies` 这一行——如果没有它，并行编译可能导致可执行文件在消息代码生成完成之前就开始编译，产生莫名其妙的头文件找不到错误。

### 7. 编译验证

```bash
cd ~/catkin_ws
catkin_make
source devel/setup.bash

# 验证消息是否注册成功
rosmsg show custom_msgs/Complex
# 输出：
# float64 real
# float64 imag
```

## 消息中的 Header

几乎所有传感器消息都包含一个 `std_msgs/Header` 字段：

| 字段 | 含义 |
|---|---|
| `seq` | 消息序列号（自增） |
| `stamp` | 时间戳（ros::Time） |
| `frame_id` | 坐标系名称（如 "base_link"、"camera_link"） |

C++ 中填充 Header：

```cpp
#include <std_msgs/Header.h>

std_msgs::Header h;
h.seq = seq_counter++;
h.stamp = ros::Time::now();
h.frame_id = "base_link";
```

## 类型嵌套

消息可以嵌套其他消息类型（包括自定义的）。例如定义一个圆的消息：

`msg/Circle.msg`：
```
float64 radius
geometry_msgs/Point center
custom_msgs/Complex rotation
```

在代码中使用：
```cpp
custom_msgs::Circle c;
c.radius = 1.0;
c.center.x = 0.0;
c.center.y = 0.0;
c.center.z = 0.0;
c.rotation.real = 1.0;
c.rotation.imag = 0.0;
```

## 数组与定长数组

```
int32[] values          # 不定长数组
float64[10] fixed_arr   # 定长数组（10 个元素）
string[] names          # 字符串数组
```

C++ 中操作：
```cpp
msg.values.push_back(42);
msg.values.size();       // 获取数组长度
```

## 常见错误

| 错误 | 原因 |
|---|---|
| `fatal error: custom_msgs/Complex.h: No such file or directory` | 没加 `add_dependencies`；或忘记先 `catkin_make` |
| `rosmsg show` 找不到消息 | 忘记 `source devel/setup.bash` |
| 编译报 `Unknown CMake command "add_message_files"` | CMakeLists.txt 里 `find_package` 没加 `message_generation` |
| `package.xml` 中没加 `message_generation` | 执行 `catkin_make` 时不会生成消息代码 |

---

## 一句话总结

**自定义 `.msg` 文件 → 配置 CMakeLists.txt 四个关键步骤 → ROS 自动生成 C++/Python 类 → 在节点中像使用普通 struct 一样使用消息，这是 ROS 语言无关消息系统的核心机制。**
