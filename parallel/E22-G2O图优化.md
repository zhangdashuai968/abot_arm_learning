# E22 · 基于 G2O 的图优化

**阶段**：Phase 3
**预计耗时**：3-4 小时
**前置**：E21（闭环检测体感）
**对应官方实验讲义**：`~/桌面/abot开源机器人/v01.01.01.0000/实验22_基于G2O激光SLAM图优化实验.md`

## 目标

理解 SLAM 后端"图优化"的本质：**节点 = 位姿 / 路标，边 = 测量约束**，整个 SLAM 问题转化为最小化所有边的残差。能跑一遍 g2o 自带的 demo 看可视化。

## 关键问题

1. 图优化的"图"和数据结构里的"图"是同一个吗？节点和边代表什么？
2. 边的"信息矩阵"是什么？等价于协方差矩阵的什么？
3. g2o vs ceres vs gtsam，三大优化库各自的优势场景？
4. 后端优化耗时随节点数增长是 O(n²) 还是 O(n³)？为什么？（提示：稀疏求解）
5. 闭环边给错了（错配），整张图会怎样？有什么办法防御？（鲁棒核函数）

## 执行步骤

### 1. 跑 g2o demo
```bash
sudo apt install g2o-cli
g2o_viewer    # 加载 .g2o 文件可视化
```
官方有几个 toy example（如 manhattan world），加载看看。

### 2. 把 E21 的 SLAM 输出导出成 .g2o
- 多数 SLAM 包都能 dump pose graph
- 用 `g2o_viewer` 打开看看节点与边的分布

### 3. 手动改一条边权重
- 把闭环边的信息矩阵权重调小 / 调大
- 重跑优化看地图变化

### 4. 理解一段代码
读 `karto_slam` 里调 g2o 的部分（约 50 行），写一段注释说明每行干什么。
