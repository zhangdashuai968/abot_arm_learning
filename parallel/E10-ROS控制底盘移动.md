# E10 · ROS 控制底盘移动

**阶段**：Phase 2
**预计耗时**：2-3 小时
**前置**：Phase 1 完成（ROS 基础 + E02 系统已摸过）
**对应官方实验讲义**：`~/桌面/abot开源机器人/v01.01.01.0000/实验10_ROS下控制机器人移动实验.md`

## 目标

能用 `/cmd_vel` 控制 Zeus S2 底盘走出"直线 / L 形 / 原地转 90°"，并测出实际位移与命令值的误差。

## 关键问题（执行前先尝试用直觉回答）

1. `/cmd_vel` 消息类型是什么？`linear.x` 单位是 m/s 还是 cm/s？
2. 四轮差速转向的运动学约束：`linear.y` 在差速车上能用吗？给非零会怎样？
3. 厂家底盘控制器是不是直接订阅 `/cmd_vel`？还是有中间话题？
4. 最大线速度 / 角速度限位在哪个文件配置？超限后是裁剪还是报错？
5. 用 `rosservice` 让车原地转 360° 与用 `/cmd_vel` 持续发送，哪个更精确？为什么？

## 执行步骤

### 1. 找到底盘控制接口
```bash
rostopic info /cmd_vel              # 看消息类型与发布/订阅者
rosnode info /<底盘控制器节点名>     # 看它订阅了哪些 topic
```

### 2. 手动遥控验证（**车下垫起来 / 轮子离地**）
```bash
rosrun teleop_twist_keyboard teleop_twist_keyboard.py
```

### 3. 用代码走路径
写一个 Python 节点 `square_walker.py`：
- 直线 1 m → 原地 90° → 直线 1 m → 90° → ...（走出 1m × 1m 方框）
- 用 `/odom` 计算实际走了多远，与命令对比

### 4. 参数与限速
- 找到底盘最大速度配置（猜在 `<bringup>/config/base.yaml`）
- 改一个参数（比如最大线速度从 0.5 → 0.3 m/s），看实际响应
